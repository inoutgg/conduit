# Dumping a Database Schema

The `conduit dump` command extracts the full schema of a live PostgreSQL
database and outputs it as DDL statements.

## Usage

```sh
conduit dump --database-url "postgres://user:pass@localhost:5432/mydb"
```

Or with the environment variable:

```sh
export CONDUIT_DATABASE_URL="postgres://user:pass@localhost:5432/mydb"
conduit dump
```

The output is written to stdout, so redirect it to a file:

```sh
conduit dump > schema.sql
```

## What gets dumped

The dump includes all user-defined objects in the `public` schema: tables,
columns, constraints, indexes, types, and sequences. The output is annotated
with the conduit version that produced it:

```sql
-- Code generated by conduit via pg-schema-diff. DO NOT EDIT.
-- conduit: v0.1.0

CREATE TABLE users (
    id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    email text NOT NULL,
    created_at timestamp with time zone DEFAULT now() NOT NULL
);
```

## Common use cases

### Bootstrapping a new project from an existing database

If you have a database that was not managed by conduit, dump its schema and use
it as the target schema file:

```sh
conduit dump > schema.sql
conduit init
conduit create diff initial_schema --schema schema.sql
```

The generated migration will contain the DDL needed to recreate the current
database state. You can then mark it as already applied (since the database
already has this schema) and start iterating from there.

### Inspecting the current state

Use `dump` to get a quick snapshot of what the database looks like right now,
without connecting to it through a SQL client:

```sh
conduit dump | less
```
